# Global Conventions

## When to use this skill:

- When naming new files, components, or folders
- When deciding on model/controller/view naming conventions
- When choosing database table and column names
- When organizing code into the correct directories
- When following MVC architectural patterns
- When creating new routes and their corresponding resources
- When deciding singular vs plural naming (Model vs Controllers)
- When structuring new features or modules
- When following RESTful action naming (index, show, new, create, edit, update, delete)
- When maintaining consistency with existing codebase patterns

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.
