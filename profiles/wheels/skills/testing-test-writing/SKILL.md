# Testing Test Writing

## When to use this skill:

- When creating unit tests for model validations and methods
- When writing integration tests for controller actions
- When implementing end-to-end tests for user workflows
- When working on files in `tests/` or `specs/` directories
- When writing TestBox BDD-style specs (describe, it, expect)
- When implementing setup() and teardown() methods for test fixtures
- When testing model associations and callbacks
- When testing API endpoints and JSON responses
- When mocking dependencies or external services
- When debugging failing tests or improving test coverage

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle testing test writing.
