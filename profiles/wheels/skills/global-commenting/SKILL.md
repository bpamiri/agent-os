# Global Commenting

## When to use this skill:

- When adding documentation comments to functions and methods
- When writing inline comments to explain complex logic
- When documenting model associations and their purposes
- When explaining migration changes or database schema decisions
- When adding @param and @return documentation to functions
- When commenting controller action behavior or filter logic
- When documenting configuration options or settings
- When explaining why code exists, not just what it does
- When deciding between adding a comment vs refactoring for clarity
- When writing file-level or module-level documentation

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global commenting.
