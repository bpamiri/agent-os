## View Specifications

### 1. Index View (`pages/index.html`)

**Purpose**: Display a filterable, sortable list of items with quick actions.

**User Goals**:
- Quickly scan items to find what they need
- Filter by status or search by name
- Take action on items (view, edit, delete)

**Layout Structure**:
```
┌─────────────────────────────────────────────┐
│ Header: Title + Create Button               │
├─────────────────────────────────────────────┤
│ Filters: Search | Status Dropdown | [Clear] │
├─────────────────────────────────────────────┤
│ Table                                       │
│ ┌─────┬────────┬────────┬─────────┬───────┐ │
│ │ ✓   │ Name   │ Status │ Created │ ••• │ │
│ ├─────┼────────┼────────┼─────────┼───────┤ │
│ │     │ Item 1 │ Active │ Jan 12  │ ••• │ │
│ │     │ Item 2 │ Pending│ Jan 11  │ ••• │ │
│ └─────┴────────┴────────┴─────────┴───────┘ │
├─────────────────────────────────────────────┤
│ Pagination: < 1 2 3 ... 10 >                │
└─────────────────────────────────────────────┘
```

**Components Used**:
- `table` with `table-zebra` modifier
- `badge` for status (see `_status-badge.html`)
- `pagination` (see `_pagination.html`)
- `dropdown` for row actions
- `input` with search icon for filtering

**States**:
- Default: Table with data
- Empty: Show `_empty-state.html` component
- Loading: Table skeleton or spinner
- Error: Alert banner above table

**Responsive Behavior**:
- Desktop: Full table with all columns
- Tablet: Hide "Created" column
- Mobile: Card-based layout (optional) or horizontal scroll

## Reusable Components

### `_status-badge.html`
Maps status values to appropriate badge styles:
- `pending` → `badge-warning` (yellow)
- `active` → `badge-info` (blue)
- `completed` → `badge-success` (green)
- `cancelled` → `badge-error` (red)
- default → `badge-ghost` (gray)

### `_empty-state.html`
Displays when collections are empty:
- Icon (customizable)
- Heading: "No [resources] found"
- Description: Context-specific message
- CTA Button: Links to create action

### `_pagination.html`
Standard pagination with:
- Previous/Next buttons
- Page number buttons
- Current page indicator
- Total count display

### `_delete-modal.html`
Confirmation dialog for destructive actions:
- Warning icon
- Confirmation message
- Cancel and Confirm buttons
- Loading state during deletion

## Interaction Patterns

### HTMX Integration (if applicable)

**List Filtering**:
```html
<input 
  type="search"
  hx-get="/items"
  hx-trigger="input changed delay:300ms"
  hx-target="#items-table"
  hx-swap="innerHTML"
/>
```

**Inline Status Update**:
```html
<select
  hx-put="/items/{id}/status"
  hx-target="closest tr"
  hx-swap="outerHTML"
>
```

**Delete with Confirmation**:
```html
<button
  hx-delete="/items/{id}"
  hx-confirm="Are you sure you want to delete this item?"
  hx-target="closest tr"
  hx-swap="outerHTML swap:1s"
>
```

## Accessibility Notes

1. **Form Labels**: All inputs have associated labels
2. **Error Messages**: Connected via `aria-describedby`
3. **Focus Management**: Modal traps focus, returns on close
4. **Keyboard Navigation**: All actions accessible via keyboard
5. **Color Contrast**: Status badges include icons, not just color
6. **Screen Readers**: Tables use proper `<th>` scope attributes

## Implementation Guidance

### For CFML/Wheels
- Use `#urlFor()#` for URL generation
- Use `#errorMessageOn()#` for validation errors
- Use `#textField()#`, `#select()#` helpers with DaisyUI classes

### For Rails/ERB
- Use `<%= link_to %>` and `<%= form_with %>` helpers
- Use `<%= @item.errors[:field] %>` for validation
- Partials map to `components/_*.html.erb`

### For Laravel/Blade
- Use `{{ route() }}` for URLs
- Use `@error` directive for validation
- Components map to Blade components
